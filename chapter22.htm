22.1.     Overview

Machine Learning (ML) Tool Kit in the i2G platform is an advanced tool that offers many state-of-the-art algorithms in the industry. They are divided into 2 groups, namely Regression and Classification.

Fundamentally, classification is about predicting a label (discrete properties such as facies) and regression is about predicting a quantity (continuous properties such as permeability).

Before running any ML technique, the user is required to create a ML model:

Step 1.         Go to “Machine Learning” tab

Figure 22‑1 Machine learning tab on Menu Bar

Step 2.         Select the “Regression Model” or “Classification Model” to make a new model. If the user wants to open an existing model, select “Open Model”

Step 3.         Fill in “Model name” if a new model is being created

Figure 22‑2 New model dialog

22.2.3.         Model Parameters – Multi Perceptron

       Multi Perceptron is a class of feedforward artificial neural network. An MLP consists of three types of layers of nodes, including an input layer, hidden layers, and an output layer. Except for the input nodes, each node is a neuron that uses a nonlinear activation function. MLP can be used for both classification and regression problems. The supervised learning technique utilized for MLP is the backpropagation algorithm

Figure 22‑8 Select Multi Perceptron model

Input parameters

Input parameters for this model are defined as follows:

Activation: is the name of the activation function(s) for hidden nodes

·      ‘Logistic’, the logistic sigmoid function, returns f(x) = 1 / (1 + exp(-x)).

·      ‘Tanh’, the hyperbolic tan function, returns f(x) = tanh(x).

·      ‘Relu’, the rectified linear unit function, returns f(x) = max(0, x), Maximum Iteration: is the maximum number of iterations. Unless the training model converges at some point during the training process, which is determined by “Tol” parameter, it will stop when the number of iterations exceeds Max_iter.

Figure 22‑9 Multi Perceptron model information

·      Tol: Tolerance for optimization. When the loss or score is not improving by at least two consecutive iterations, convergence is considered to be reached and training stops.

·      Initial Learning Rate: It controls the step-size in updating the weights.

Tip: Advanced parameters that can be adjusted in case the normal ones are not capable of generating the desired output

Network Structure

Users can design their model architecture by typing the number of hidden layers (“Hidden_layer_sizes”) and the number of nodes in a layer (“layer 0”/”layer 1”) depending on how complex their data are.

The model architecture is visualized by the window on the right and is automatically updated when changed.

Figure 22‑20 Parameters in Classification model

Neural Network Classifier:

Neural Network Classifier is a multilayer perceptron network for classification problems. This model optimizes the log-loss function using LBFGS or stochastic gradient descent. Input parameters for this model are defined as follows.

·      Activation: is the name of the activation function for hidden layers.

·      Algorithm: is the name of the training algorithm, which is either a backpropagation or evolution strategy algorithm.

·      The backpropagation algorithm works best with many epochs.

·      The evolution algorithm is preferable with a smaller number of epochs, about 1000.

·      Batch_size: Number of samples per gradient update. If batch_size =0, the number of training samples will be used.

·      Learning_rate: It controls the step-size in updating the weights.

·      Num_epochs: maximum number of training iterations.

·      Optimizer: is the solver for weight optimization.

·      Warm-up: is to determine the initial values of weights. When set to True, the model will run with batch_size = 1 and num_epochs = 1 to get the initial weights, otherwise, it just randomizes the initial weights. After this process, the model will run the training process as set by other parameters.

·      Boosting_ops: this parameter is only used for the evolution algorithm. This parameter determines the number of times that the weights are further updated based on the accuracy after the training process based on the loss function finishes. The default is 100.

·      Sigma: this parameter is only used for the evolution algorithm. This parameter determines the variance size of the Gaussian distribution used for randomizing the weights.

·      Population: is the number of sets of randomized weights for each epoch. This parameter is only used for the evolution algorithm.
