Module 17:  Workflow Manager

A Workflow is a tool for displaying and modifying several computational Tasks diagrams on i2G. It defines a task model for graphical elements and their relationship. The Workflow allows you to work in a multi-well and/or multi-zone environment, while efficiently controlling interpretation parameters. Additionally, Workflow allows users to connect the methods of calculation together to indicate which output of one method will be the input of another method.

17.1.     Create a new workflow

Step 1.         At the bottom of Data Tree, right-click on Workflow and choose New Flow from the contextual menu. The New Workflow dialog pops up.

Figure 17‑1 Add a new workflow

Tip: you can also open the New Workflow dialog by choosing the Workflow Manager in Home tab, then click on Add () icon from the Open Workflow pop-up dialog

Step 2.         Enter a name for the new workflow

Step 3.         Choose a predefined workflow template or skip it if you want to create a blank workflow (check out Workflow Template for more information)

Step 4.         Click Ok.

17.2.     Open an existing workflow

Step 1.         From the Home tab, choose the Workflow Manager. The Open Workflow dialog pops up.

Figure 17‑2 Open an existing workflow

Step 2.         Select a workflow from the dropdown box

Step 3.         Click Ok

17.3.     Add a task to a workflow

Step 1.         At the top right corner of the Task window, hover over the Task button. The contextual menu appears.

Figure 17‑3 Add a Task to Workflow from the Task window

Step 2.         Choose Add Task to Workflow from the contextual menu, the Open Workflow dialog pops up.

Step 3.         Choose an existing workflow from the dropdown box

Step 4.         Click Ok. The selected workflow will automatically open next to the current Task window.

Step 5.         In Canvas, a new Task event is highlighted in orange. Hover the pointer to anywhere you want to put this Task into the workflow, and then left-click to end the process, the task color will change from orange to black.

Figure 17‑4 Put a Task into the Canvas

17.4.1.1.    Tool Bar

At the top left corner of the workflow window, you will see the Tool Bar, where you can access a range of actions to the diagram.

Name of Icon

Save Workflow after changing

17.4.1.2.    Workflow’s name

Click on this box to rename your workflow

17.4.1.3.    Data List

When a Task is added into the workflow, it will be listed on the left side panel of the workflow. All the Wells and the Zone Sets that are available in this Task will be displayed as well.

Choose an Output Dataset to store all output curves from the activated Tasks.

Figure 17‑6 Data List in the Workflow

Tip: To deactivate a Task, click on the Task name in the list

17.4.1.4.    Canvas

The Canvas is the space in which you will create your diagrams. Hold down the Ctrl key while you are scrolling to zoom in and out the Canvas.

Figure 17‑7 Canvas in the Workflow

17.4.1.5.    Palette

On the top left of the Canvas, click (  ) button to expand the Palette. It gives you quick access to a series of tools each designated by an icon.

Table 17‑2 Palette in the Workflow

Icon

Name of Icon

Description

Hand tool

Hand Tool helps you to freely navigate the canvas without accidentally moving elements under the cursor

Lasso tool

Lasso Tool allows you to easily select a number of diagram elements. You can then proceed to move these selected elements onto valid locations on the Canvas. Activate the Lasso Tool from the Palette or hold down CTRL + drag using the mouse

Space tool

Space Tool allows you to create or remove space both horizontally and vertically in a diagram. If you would like to remove space instead of creating it use the CTRL modifier (ALT on Mac) during initialization

Connect Line

The Line, being essential to diagramming, connects flow Tasks in proper sequential order on Canvas

Parallel Gateway

Parallel Gateway is used to represent two concurrent tasks in a workflow. It’s not dependent on conditions or events

The tools can be activated via the hand tool icon on the Palette. To deactivate it, simply click anywhere or press ESC on your keyboard.

After adding a Task to your workflow, you can drag and drop to move them from one place on the Canvas to another.

Click on Task event to choose an action from its context menu

Figure 17‑9 Actions for a Task event

17.4.3.         Work with Lines

The Line, being essential to diagramming, connects flow Tasks in proper sequential order on Canvas.

17.4.3.1.    Add Lines to diagram

Step 1.         Focus on a Task, select ( ) and drag your cursor away from the Task. A line connected to an arrow will extend from the Task as you drag

Tip: you can also activate the Line mode from the Palette

Step 2.         When you reach the border of the second Task, hold down your cursor until the Task is highlighted in green and a red outline appears on the Task’s border

Step 3.         Release your cursor, the second Task returns to white

Note:

-       The Lines can only be created when you already have Tasks on your Canvas

-       If you move a Task that is connected with a line to another Task, the line will automatically adjust its connection points to form the most efficient path between the two Tasks

17.4.3.2.    Modify Line path

Step 1.         Select a line by clicking on it. A yellow dot will appear in the corner of each leg. If you hover over a section of the line, a lighter yellow dot will appear at the cursor within that leg.

Figure 17‑10 Select a line between 2 Tasks

Step 2.         Click and drag one of these yellow dots to change the path of that section of the line

Figure 17‑11 Modify Line path

Tip: Focus on a line and select ( ) to delete the line between 2 tasks
