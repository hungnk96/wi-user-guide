Module 14:  Project Storage

14.1.     What is Project Storage

Project Storage is a cloud storage that allows users to store files/folders on their project in i2G. Project Storage will be shared with particular users having permission to access its project.

To open Project Storage: From Menu Bar, click on Home tab and choose Project Storage, the Project Storage window opens.

Figure 14‑1 Open Project Storage window

14.2.     Files you can store in Project Storage

Any file type can be stored in Project Storage:

  • General files:
    • Images files: (.JPEG, .PNG, .GIF, .BMP, .TIFF, .SVG)
    • Audio formats: (MP3, MPEG, WAV, .ogg, .opus)
    • Video files: (WebM, .MPEG4, .3GPP, .MOV, .AVI, .MPEGPS, .WMV, .FLV, .ogg)
    • Archive files:  (.ZIP, .RAR, tar, gzip)
    • Markup/code: (.CSS, .HTML, .PHP, .C, .CPP, .H, .HPP, .JS, .java, .py)
    • Text files: (.TXT)
    • Las files: (.LAS)
  • Microsoft Office files:
    • Excel (.CSV, .XLS and .XLSX)
    • PowerPoint (.PPT and .PPTX)
    • Word (.DOC and .DOCX)
    • XML Paper Specification (.XPS)
  • Adobe files: .AI; .PSD; .PDF; .SVG; .TIFF; .TTF
  • …

14.3.     Metadata

Each file/folder stored on your storage has metadata associated with it. Metadata is the information of a file’s source, size, rating, last modified date, and more. This metadata makes searching and organizing files on storage a lot faster.

When you select a file/folder, its metadata will be displayed in the right bar.

Figure 14‑2 Metadata of an object in Project Storage

There are 4 types of metadata in i2G:

·      General: basic metadata that describes a resource for identification purposes such as File Name, Type, Size, Location, Source, Author, Date Uploaded, Date Modified. These metadata elements are created by automated information processing so the user cannot edit it except the File Name.

·      Information: provide information to help users manage the resource, describe content and quality of data, track the relationship between files and objects on data tree. There are 6 metadata elements in this class:

§  Well, Field, Well Type: the particular resource the file/folder belongs to

§  Data Type: type of data in file/folder such as Log, Routine Special Core, MDT, Mud Log…

§  Quality: quality of file data

§  Associate to: view objects on data tree that was associated to this file/folder

·      Description: users can summarize information of the file/folder

·      More Information: allow users to input any metadata which is relevant or necessary to describe the file/folder.

Click Add Metadata button, the Add Metadata dialog pops up. Then enter a Name and Value, click Ok to add the new metadata for that file/folder.

Figure 14‑3 Add Metadata dialog

14.4.     Upload files and folders to Project Storage

Step 1.         Click Upload () icon at the top left, Upload Files dialog pops up.

Figure 14‑4 Upload Files dialog

Step 2.         Click Select Files/Folders > choose the files/folder you want to upload then click Open.

Step 3.         Add metadata for items. Keep checking the Bulk Edit option if you want to update metadata for multiple items at once, if not, uncheck this one and modify each item manually.

Tip: you can select files/folders for many times at 1 working phase. New selected files/folders will be added right under the previously selected one.

Step 4.         Click Upload after you define all the options.

Note: If you upload a file/folder with the same name, i2G will hold this one. You need to choose Overwrite or Remove from Upload List option. If you want to overwrite all the existing files/folders, click Overwrite All.

14.6.     Search for files/folders

Sometimes remembering precisely where you stored a file might be difficult.  I2G allows you to find them more easily by searching for the file name or file metadata.

14.6.1.         Simple search by filename

Step 1.         At the top right, type a word or phrase into the Search box

Step 2.         Press Enter. Search results will appear as you type

Figure 14‑5 Simple Search box

Step 3.         To clear and close the search, click the Close button ( ) next to the Search box or ( ) next to the FILTER: CUSTOM SEARCH MODE

14.6.2.         Advanced search by file metadata

When you cannot find the files or folders you’re looking for by using a simple Search box, you can perform an advanced search that gives you more options to find them easier.

Step 1.         Click on Advanced Search ( ) button next to the Search box. The Advanced Search dialog pops up.

Figure 14‑6 Advance Search dialog

Step 2.         Choose the kind of item you want to search for. There are 3 options: File, Folder and All (both File & Folder).

Step 3.         Check/uncheck Sub-folders to include/exclude subfolders in your search.

Step 4.         Select metadata you want to search for File Name, File Type, Author, Data Uploaded, Field, Well, Well Type, Data Type, and Custom. Then type in search criteria in the box or select from the available criteria.

Tip: You can combine multiple metadata at once to narrow your search results, for example:

Figure 14‑7 Narrow search results using Advance Search

Note: Custom element is user-defined metadata that was added to the item before.

Step 5.         To clear and close the search, click the Close button ( ) next to the Search box or ( ) next to the FILTER: CUSTOM SEARCH MODE

14.7.     File Viewing

I2G lets you preview files from within the project storage without having to download them, first.

Figure 14‑8 Preview file in Project Storage

The i2G viewer allows the following file formats to be viewed:

·      Adobe Portable Document Format (.PDF)

·      Image files (.JPEG, .PNG)

·      Text files (.TXT)

·      Comma Separated Values files (.CSV)

·      Las files (.LAS)

·      Markup/Code (.HTML, .JS)

·      Python (.PY)

·      …

Note: If your files are in other formats that cannot be viewed, i2G gives you the option to download it to your desktop.

14.8.     Relationship between Project Storage and Data tree

I2G allows users associate files/folders to any object on data tree such as well, dataset, curve, zone, marker, plot, workflow... to help users manage resources and track the relationship between them easier.

14.8.1.         Create a relationship from files/folders to data tree

Step 1.         In Project Storage, focus on a file/folder you want to link > on the right metadata bar, click Copy Location ( ) icon next to the Location box. This one icon turns red.

Figure 14‑9 Copy Location of a file in Project Storage

Tip: in case you want to link for multiple files/folders at once, Copy Location ( ) icon will be located on the top navigation bar instead.

Step 2.         On data tree, focus on an object > in the Properties tab, click Paste ( ) icon next to the hyperlink line to completely link this object with the selected files/folders.

Figure 14‑10 Paste file's location to an object on Data tree

Tip: to add/remove a link from an object on data tree, click the Add () / Remove () button on the right side of each hyperlink line

14.8.2.         Create a relationship from well on data tree to a file/folder

Step 1.         On data tree, focus on a well > in the Properties tab, click Copy Reference () icon. This one icon turns red.

Figure 14‑11 Copy Reference of a well on Data tree

Step 2.         In Project Storage, focus on a file/folder you want to link to the selected well > in the right metadata bar, click Paste () icon next to Well box. Now you can see the Well name appears in this box, the Field and the Well Type will also appear in its box if it were defined in the well header.

Figure 14‑12 Paste well's reference to a file/folder in Project Storage

14.8.3.         Associate Well on Data tree with File/Folder

Step 1.         On data tree, focus on an object

Step 2.         In Properties tab that shows properties of an object, click Go to files ( ) icon on the right side of the first hyperlink line. Filter status turns Related Mode which means Project Storage will only display files/folders that were associated with this object.

Figure 14‑13 Associate Well on Data tree with File/Folder

14.8.4.         Associate File/Folder with Data tree

Step 1.         On Project Storage, focus on a file/folder

Step 2.         In the right metadata bar, click the Associate () icon. Filter status turns File Mode that means the Data tree will only display objects that were associated with this file/folder.

Figure 14‑14 Associate File/Folder with Data tree
