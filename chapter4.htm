Module 4:       Data Import

4.1.        Inventory Explorer

Importing files into Project comprises two phases:

o   Import from desktop to Inventory Explorer: The files will be loaded and patched into Inventory Explorer and ready to be shared for different projects in the later step.

o   Import from Inventory Explorer to Project: load wells from Inventory Explorer to any projects.

Note: If you can't import data, the owner may have disabled option to import

4.1.1.   Import data from the desktop to Inventory Explorer

4.1.1.1.        Import *.las files

Figure 4‑1 Import *.las from desktop to Inventory Explorer

Step 2.       In the Inventory Explorer window, select Import LAS from the dropdown list

Step 3.         Select one or multiple Las files, Click Open

     The wells will be imported into Inventory Explorer

4.1.1.2.        Import *.csv file

Step 1.         From Menu Bar, click on Data tab > click Import and select Inventory Explorer, the Inventory Explorer window will be displayed

Figure 4‑4 Import *.las from desktop to Inventory Explorer

Figure 4‑5 Import Csv/Ascii Wizard dialog

Step 3.         Users can use “drag and drop” *.csv file from the computer or click on Drop/Click to upload to select the file from the window browser. Then click Open.

Figure 4‑6 Browse and navigate to the *.csv file location

Step 4.         User has to define a data file structure by selecting the line number containing:

·      Set the Header line (mandatory)

·      Set Unit line

·      Set Data line (mandatory)

There are 2 ways to set those lines: Enter a number into the box or pick directly on the data line. Confirm by choosing Next.

Figure 4‑7 Define data lines for *.csv file

 Tip: Users can choose the number of lines to be shown in the preview window. 50 lines from the top of the file will be set as default.

Step 5.         In the *.csv-Wizard window, define Reference Column by entering a number into the box or picking directly on the data column. By default, the first column will be automatically set as Reference Column.

Figure 4‑8 Define data columns for *.csv file

Note:

-       Well name and data set name will be automatically defined based on the original file names

-       Default Null value is -9999

-       Depending on file types, the position of the depth column may be various. Therefore, users need to manually correct it.

-       It is not necessary to define the curve columns, I2G will identify them automatically.

Step 6.         Click Load. Data will be uploaded onto Inventory Explorer

4.2.        Import Image

  This procedure shows how to load core images into i2G.

Step 1.         From Menu Bar, click on the Data tab and choose Image Manager, the Image Manager window will be displayed.

Step 2.         In the Image Set Manager window, click the Import button. The Image import wizard dialog opens.

Figure 4‑17 Open Import Image window

Step 3.         Click Select Files and navigate to your image folder. Choose all the images you want to load and click Open.

Figure 4‑18 Browse and navigate to the image file's location

Note: i2G allows you to import *.png, *.jpg image formats.

 

Step 4.         At the Name Model box, type the name or select a predefined pattern (%WELLNAME%; %TOPDEPTH%; %BOTDEPTH%). The name of the images file must comply with the naming convention for images.

Figure 4‑19 Set parameters for the image files

For example, if your image is named:

·      3334-3335: the naming convention is: %TOPDEPTH%BOTDEPTH%

·      ESS_3334: the naming convention is: %WELLNAME%TOPDEPTH%

 

 Tips:

-       Users can import images of multiple wells at the same time by using pattern %WELLNAME% in the name model. If the well’s name in the image file name matches the one in the data tree, the name turns green, if not, it turns red.

-       Users can preview the image by clicking the Preview button on the right side of each image.

 

Step 5.         The left-hand side table shows you how the images are stored.

·      Choose a Well (this box is ignored when %WELLNAME% already exists in the Name Model)

·      Enter an Image Set name

·      Choose a proper Unit

·      Enter a Maximum Interval value. This option is ignored when the top and bottom are available. If only tops are available, a check is run on the difference between them and the maximum interval is chosen. If the difference between tops is longer than the maximum interval, the core length will be equal to the maximum interval chosen to prevent the core image from being stretched between the two tops.

Step 6.         Click Upload to launch the import procedure after you defined all the options. The Image set is now added below well in the data tree.                                              

 

 

4.3.        Import Zone

Step 1.         From Menu Bar, click on Data tab and choose Zone Set Manager, the Zone Set Manager window will be displayed

Figure 4‑20 Open Zone Set Manager window

Step 2.         In the Zone Set Manager window, select the Import Zone Set (  ) icon, the Import Zone Set dialog appears. Drag-and-drop zonation files from the computer into the window or select Drop/Click to the uploaded button to select the file from the window browser. Then click Open.

Figure 4‑21 Import Zone Set dialog

Figure 4‑22 Browse and navigate to the Zone file's location

Step 3.         Users must define data lines, including:

•       Header line (mandatory input): name of the input curves

•       Unit line (not mandatory): Units of inputs

•       Data first line (mandatory input): the first line where data is available

  There are 2 ways to set those lines: Enter a number into the box or pick directly on the data line. Confirm by choosing Next.

Figure 4‑23 Define data lines for zone file

Step 4.         The software will determine the Zone Name by the original filename. Set up a column:

•       Well column (mandatory): the first column is set as default.

•       Zone column (mandatory): the second column is set as default.

•       Top column (mandatory): the third row is set as default.

•       Bottom column (mandatory): the fourth row is set as default.

Figure 4‑24 Define data columns for zone file

Tips:

-       Depending on the file type, the position of the column may be different. Therefore, the user needs to correct it manually

-       Users can edit unit in the unit line

Step 5.         In the Zone Set tab, all available zones in the well will be listed. In the Template set tab, for each imported zone set, a zone template set will be automatically generated and listed. Each of them contains the setting of that Zone Set.

Figure 4‑25 Available Zone Set in the Well

Figure 4‑26 A Zone Template Set will be automatically generated when importing zone set

    Note:

-       The zone can only be imported into the currently available wells in the project

-       Zone depth must not exceed the well depth.

-       The zone must not overlap on each other
