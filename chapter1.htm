Module 1:       I2G Administrators

Users with the Company Admin permission can perform all administration tasks, including managing users across multiple projects and applications in i2G platform; manage the group of users; assign users to groups, project roles; share projects and monitor a user’s activities.

1.1.        Getting started

1.1.1.   Log into i2G administrators site

Step 1.         Go to https://admin.i2g.cloud/. Only users with the Company Admin permission can access this site.

Step 2.         At the Sign In the dialog, enter your Username and Password.

Figure 1‑1 log into i2G-admin site

Step 3.         Click Sign In.

If you’re having trouble logging in, please contact your administrator or contact us via i2g.cloud/contact-us

To log out of site: Click   icon at the top right corner.

1.1.3.   User management

By default, when you navigate to the admin site, you will be redirected to the User page first. You can also open this page by clicking ( ) on the Navigation Bar

From here you can view and edit existing operators as well as adding new ones. When editing an operator, you can change all the details of the operator including resetting the password

Figure 1‑3 User management tab

1.1.3.1.        Add new users

You can add users directly to your company following these steps:

Step 1.         From the User page, click  button on the top Tool Bar. The New User dialog pops up

Figure 1‑4 Add new users dialog

Step 2.         In the New User dialog, specify general information of the new user, following:

·      Username

·      Full name

·      Password

·      Email

·      License: Give users access to different modules in i2G such as Full feature, Base, Clastic, CODB, Python …

·      Role: fine-tune exactly what each of your users can do in i2G platform

o   Company Admin: can perform all administration tasks, including manage users across multiple projects and applications in i2G platform; manage group of users; assign users to groups, project roles; share projects and monitor a user’s activity

o   Normal User: can access https://wi.i2g.cloud/ only.

o   Company Storage (Full Permission): can access both https://wi.i2g.cloud/ and https://storage-admin.i2g.cloud/ with Full permission in CODB.

o   Company Storage (Upload only): can access both https://wi.i2g.cloud/ and https://storage-admin.i2g.cloud/ with only Upload permission in CODB

o   Company Storage (Edit only): can access both https://wi.i2g.cloud/ and https://storage-admin.i2g.cloud/ with only Edit permission in CODB

o   Company Storage (View only): can access both https://wi.i2g.cloud/ and https://storage-admin.i2g.cloud/ with only View permission in CODB

·      Status: Activate or Deactivate user who no longer need to use the system

Tip: You can always change the user information later by clicking  on the top of Navigation Bar

Step 3.         Click Ok to add the new user after you define all the information

1.1.4.   Group management

When multiple users in your site need the same permissions or restrictions, you can add the users to a group and then give the group product access.

Click (  ) on the Navigation Bar to open Group management

1.1.4.1.        Add new groups

Step 1.         From the Group page, click  button on the top Tool Bar. The Add New Group dialog pops up

Step 2.         In the Add New Group dialog, specify general information of the new group such as Name and Description.

Tip: You can always change the group’s information later by clicking  on the top of Navigation Bar

Step 3.         Click Ok to add the new group after you define all the information

1.1.4.2.        Edit user information

Step 1.         From the Group page, focus on a desired group line > click  button on the top Tool Bar. The Group Info dialog pops up

Figure 1‑6 Edit group information

Step 2.         Click Ok after changing.

1.1.4.3.        Assign users to group

Step 1.         From the Group page, select a group where you want to add users > click  button on the top Tool Bar

Step 2.         In the Group Info dialog, focus on Members tab. A left table lists all users of your company and the right one lists all members of the selected group.

Figure 1‑7 Assign users to group

Step 3.         On the left table, click ( ) icon next to a username that you would like to add. The selected username will be added on the right table

Step 4.         Click Ok.

1.1.4.4.        Remove users from a group

Step 1.         From the Group page, select a group where you want to remove users > click  button on the top Tool Bar

Step 2.         In the Group Info dialog, focus on Members tab. A left table lists all users of your company and the right one lists all members of the selected group.

Figure 1‑8 Remove users from the group

Step 3.         On the right table, click ( ) icon next to each username

Step 4.         Click Ok.

1.1.4.5.        Delete groups

Step 1.         From the Group page, focus on a desired group line > click  button on the top Tool Bar. The Confirmation dialog pops up

Step 2.         Click Yes.

1.1.5.   Project Sharing

I2G is designed to keep your user data safe and make sure anonymous users cannot access the content that they have no permission to view. But sometimes, you need to share a project with someone who does not belong to your organization. This Project tab will help you to do that.

Click (  ) on the Navigation Bar to open Project tab

 

Figure 1‑9 Project tab in i2G-admin site

1.1.5.1.        Share a project to another organization

Step 1.         From the Project page, focus on a desired project > click  button on the top Tool Bar.

Step 2.         Now you can see a randomly generated key appearring in the Sharing Key field. Give this key to the one you’d like to share this project to.

Figure 1‑10 generate a shared project key

To open a shared project from another organization, see this article for more information

1.1.5.2.        Stop sharing a project to another organization

To stop sharing a project: from the Project page, focus on a desired project > click  button on the top Tool Bar. The generated key will be removed from the Sharing Key field.

Notes:

-       When anyone makes changes in data in a shared project, other members will immediately be able to see and use the modified data.

-       If you permanently delete a shared project or stop sharing a project key, other collaborators cannot access as well.

-        When an administrator adds to or removes members from the group, they will gain or lose permission to the projects that the group can access.
